# [300. Longest Increasing Subsequence](https://leetcode.com/problems/longest-increasing-subsequence)

## Description

Given an integer array `nums`, return _the length of the longest **strictly increasing**_ _**subsequence**_.

**Example 1:**

**Input:** nums = \[10,9,2,5,3,7,101,18\]  
**Output:** 4  
**Explanation:** The longest increasing subsequence is \[2,3,7,101\], therefore the length is 4.

**Example 2:**

**Input:** nums = \[0,1,0,3,2,3\]  
**Output:** 4

**Example 3:**

**Input:** nums = \[7,7,7,7,7,7,7\]  
**Output:** 1

**Constraints:**

- `1 <= nums.length <= 2500`
- `-104 <= nums[i] <= 104`

**Follow up:** Can you come up with an algorithm that runs in `O(n log(n))` time complexity?

## My Solution

**JavaScript**

```js
const lengthOfLIS = (nums) => {
  // Stores the smallest tails of increasing subsequences. tails[i] stores the best (smallest)
  // ending for a sequence of length i + 1.
  const tails = [];

  for (const num of nums) {
    let left = 0;
    let right = tails.length - 1;

    // Binary search to find the first element in tails that is >= num
    while (left <= right) {
      const middle = left + Math.floor((right - left) / 2);
      if (tails[middle] < num) left = middle + 1;
      else right = middle - 1;
    }

    // left now holds the index where num should be placed.
    // If left equals the length of array, num is larger than all elements,
    // so we extend the sequence.
    if (left === tails.length) tails.push(num);
    else tails[left] = num; // Otherwise, overwrite existing larger tail
  }

  // The length of the tails array is the length of the longest increasing subsequence
  return tails.length;
};
```

// Approach: Dynamic Programming - Tabulation
const lengthOfLIS = (nums) => {
  const n = nums.length;
  // Base cases: Every number can form a valid subsequence of at least length 1
  const dp = Array(nums.length).fill(1);

  for (let i = 0; i < n; ++i) {
    for (let j = 0; j < i; ++j) {
      if (nums[i] > nums[j]) {
        dp[i] = Math.max(dp[i], dp[j] + 1);
      }
    }
  }

  // The result is not necessary at the end of the dp array -- we must check!
  return Math.max(...dp);
};
```

// Approach: Brute-Force with memoization
const lengthOfLIS = (nums) => {
  const memo = Array(nums.length)
    .fill(null)
    .map(() => Array(nums.length + 1).fill(-1));

  const solve = (idx, prevIdx) => {
    if (idx >= nums.length) return 0;
    if (memo[idx][prevIdx + 1] !== -1) return memo[idx][prevIdx + 1];

    // If this is strictly greater than the previous, we can choose to take it
    const take = prevIdx === -1 || nums[idx] > nums[prevIdx] ? 1 + solve(idx + 1, idx) : 0;

    // We can always choose to not take this number
    const dontTake = solve(idx + 1, prevIdx);

    memo[idx][prevIdx + 1] = Math.max(take, dontTake);
    return memo[idx][prevIdx + 1];
  };

  return solve(0, -1);
};
```

```js
// Approach: Brute-Force (Time Limit Exceeded!)
const lengthOfLIS = (nums) => {
  const solve = (nums, idx, previous) => {
    if (idx >= nums.length) return 0;

    // If this is strictly greater than previous, we can choose to take it
    const take = nums[idx] > previous ? 1 + solve(nums, idx + 1, nums[idx]) : 0;

    // We can always choose to not take this number
    const dontTake = solve(nums, idx + 1, previous);

    // Does taking or not taking result in a longer subsequence?
    return Math.max(take, dontTake);
  };

  return solve(nums, 0, -Infinity);
};
```
