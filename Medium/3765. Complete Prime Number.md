# [3765. Complete Prime Number](https://leetcode.com/problems/complete-prime-number)

## Description

You are given an integer `num`.

A number `num` is called a **Complete Prime Number** if every **prefix** and every **suffix** of `num` is **prime**.

Return `true` if `num` is a Complete Prime Number, otherwise return `false`.

**Note**:

- A **prefix** of a number is formed by the **first** `k` digits of the number.
- A **suffix** of a number is formed by the **last** `k` digits of the number.
- Single-digit numbers are considered Complete Prime Numbers only if they are **prime**.

**Example 1:**

**Input:** num = 23  
**Output:** true  
**Explanation:**

- ​​​​Prefixes of `num = 23` are 2 and 23, both are prime.
- Suffixes of `num = 23` are 3 and 23, both are prime.
- All prefixes and suffixes are prime, so 23 is a Complete Prime Number and the answer is `true`.

**Example 2:**

**Input:** num = 39  
**Output:** false  
**Explanation:**

- Prefixes of `num = 39` are 3 and 39. 3 is prime, but 39 is not prime.
- Suffixes of `num = 39` are 9 and 39. Both 9 and 39 are not prime.
- At least one prefix or suffix is not prime, so 39 is not a Complete Prime Number and the answer is `false`.

**Example 3:**

**Input:** num = 7  
**Output:** true  
**Explanation:**

- 7 is prime, so all its prefixes and suffixes are prime and the answer is `true`.

**Constraints:**

- `1 <= num <= 10^9`

## My Solution

**JavaScript**

```js
// Approach: Using a single loop to extract prefix and suffix. Using math-based pruning
// to avoid checking numbers that have previous digits that will always prevent prefix/suffix
// from being a prime number.
const completePrime = function (num) {
  // By the definition of a prime
  if (num <= 1) return false;

  // Single digits are only vaid if they are 2, 3, 5, or 7
  if (num < 10) return num === 2 || num === 3 || num === 5 || num === 7;

  // If a multi-digit num ends in something other than 3 or 7, it is not complete prime, as it would
  // have failed as a single-digit number if it were 1, 2, 4, 6, 8, 9. This leaves 3, 5, and 7. But
  // once a number becomes mult-digit, we can eliminate 5 from this list, as that means it is a multiple of 5.
  const lastDigit = num % 10;
  if (lastDigit !== 3 && lastDigit !== 7) return false;

  // Before checking prefix/suffix, see if the full number is a prime
  if (!isPrime(num)) return false;

  // Use a single pass to check each prefix/suffix
  let prefix = Math.floor(num / 10);
  let multiplier = 10;

  while (prefix > 0) {
    let suffix = num % multiplier;

    if (!isPrime(prefix) || !isPrime(suffix)) {
      return false;
    }

    prefix = Math.floor(prefix / 10);
    multiplier *= 10;
  }

  return true;
};

// Optimized isPrime checker, checking every 6th number instead of every other.
const isPrime = (n) => {
  // Handle base cases for 2 and 3, which are by definition prime
  if (n === 2 || n === 3) return true;
  // Eliminate <= 1, evens and multiples of 3
  if (n <= 1 || n % 2 === 0 || n % 3 === 0) return false;

  // Due to the early returns above, if we start at 5 we only have to check
  // for every 6i - 1 and 6k + 1. 5 is 6(1) - 1.
  for (let i = 5; i * i <= n; i += 6) {
    if (n % i === 0 || n % (i + 2) === 0) return false;
  }

  return true;
};
```

// Approach: Brute-Force by simulating the process while avoiding string conversions
const completePrime = function (num) {
  if (num <= 1) return false;

  let prefix = num;
  while (prefix > 0) {
    if (!isPrime(prefix)) return false;
    prefix = Math.floor(prefix / 10);
  }

  let multiplier = 10;
  while (multiplier <= num) {
    let suffix = num % multiplier;
    if (!isPrime(suffix)) return false;
    multiplier *= 10;
  }

  return true;
};

const isPrime = (n) => {
  if (n === 2) return true;
  if (n <= 1 || n % 2 === 0) return false;

  for (let i = 3; i * i <= n; i += 2) {
    if (n % i === 0) return false;
  }

  return true;
};
