# [2284. Sender With Largest Word Count](https://leetcode.com/problems/sender-with-largest-word-count)

## Description

You have a chat log of `n` messages. You are given two string arrays `messages` and `senders` where `messages[i]` is a **message** sent by `senders[i]`.

A **message** is list of **words** that are separated by a single space with no leading or trailing spaces. The **word count** of a sender is the total number of **words** sent by the sender. Note that a sender may send more than one message.

Return _the sender with the **largest** word count_. If there is more than one sender with the largest word count, return _the one with the **lexicographically largest** name_.

**Note:**

- Uppercase letters come before lowercase letters in lexicographical order.
- `"Alice"` and `"alice"` are distinct.

**Example 1:**

**Input:** messages = \["Hello userTwooo","Hi userThree","Wonderful day Alice","Nice day userThree"\], senders = \["Alice","userTwo","userThree","Alice"\]  
**Output:** "Alice"  
**Explanation:** Alice sends a total of 2 + 3 = 5 words.  
userTwo sends a total of 2 words.  
userThree sends a total of 3 words.  
Since Alice has the largest word count, we return "Alice".

**Example 2:**

**Input:** messages = \["How is leetcode for everyone","Leetcode is useful for practice"\], senders = \["Bob","Charlie"\]  
**Output:** "Charlie"  
**Explanation:** Bob sends a total of 5 words.  
Charlie sends a total of 5 words.  
Since there is a tie for the largest word count, we return the sender with the lexicographically larger name, Charlie.

**Constraints:**

- `n == messages.length == senders.length`
- `1 <= n <= 10^4`
- `1 <= messages[i].length <= 100`
- `1 <= senders[i].length <= 10`
- `messages[i]` consists of uppercase and lowercase English letters and `' '`.
- All the words in `messages[i]` are separated by **a single space**.
- `messages[i]` does not have leading or trailing spaces.
- `senders[i]` consists of uppercase and lowercase English letters only.

## My Solution

**JavaScript**

```js
// Approach: Straightforward using a hash map and only a single pass.
const largestWordCount = (messages, senders) => {
  let result = '';
  let maxCount = 0;
  const wordCounts = new Map();

  for (let i = 0; i < senders.length; ++i) {
    const sender = senders[i];
    const count = messages[i].split(' ').length;

    // Update count
    const newCount = (wordCounts.get(senders[i]) || 0) + count;
    wordCounts.set(sender, newCount);

    // Is this sender now the one with the largest word count?
    // (Or tied, but sender's name is lexicographically larger?)
    if (newCount > maxCount || (newCount === maxCount && sender > result)) {
      maxCount = count;
      result = sender;
    }
  }

  return result;
};
```

| --------------- | ---------------- |
| O(N)            | O(S)             |

```js
// Approach: Straightforward using a hash map and using two loops.
const largestWordCount = (messages, senders) => {
  let result = '';
  let maxCount = 0;
  const wordCounts = new Map();

  for (let i = 0; i < senders.length; ++i) {
    const count = messages[i].split(' ').length;
    const newCount = (wordCounts.get(senders[i]) || 0) + count;
    wordCounts.set(senders[i], newCount);
  }

  for (const [sender, count] of wordCounts) {
    if (count > maxCount || (count === maxCount && sender > result)) {
      maxCount = count;
      result = sender;
    }
  }

  return result;
};
```

| Time Complexity | Space Complexity |
| --------------- | ---------------- |
| O(N)            | O(S)             |
