# [190. Reverse Bits](https://leetcode.com/problems/reverse-bits)

## Description

Reverse bits of a given 32 bits signed integer.

**Example 1:**

**Input:** n = 43261596  
**Output:** 964176192  
**Explanation:**

| Integer   | Binary                           |
| --------- | -------------------------------- |
| 43261596  | 00000010100101000001111010011100 |
| 964176192 | 00111001011110000010100101000000 |

**Example 2:**

**Input:** n = 2147483644  
**Output:** 1073741822  
**Explanation:**

| Integer    | Binary                           |
| ---------- | -------------------------------- |
| 2147483644 | 01111111111111111111111111111100 |
| 1073741822 | 00111111111111111111111111111110 |

**Constraints:**

- `0 <= n <= 2^31 - 2`
- `n` is even.

**Follow up:** If this function is called many times, how would you optimize it?

## My Solution

**JavaScript**

```js
const reverseBits = (n) => {
  let result = 0;

  // Run the loop 32 times to process all 32 bits
  for (let i = 0; i < 32; ++i) {
    // Extract rightmost bit of n
    const digit = n & 1;
    // Shift result by 1 and merge the digit to the end
    result = (result << 1) | digit;

    // Shift n right by 1 to expose the next bit
    n >>>= 1;
  }

  // Ensure JavaScript interprets the result as unsigned 32-bit
  return result >>> 0;
};
```

| Time Complexity | Space Complexity |
| --------------- | ---------------- |
| O(1)            | O(1)             |
