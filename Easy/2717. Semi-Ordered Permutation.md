# [2717. Semi-Ordered Permutation](https://leetcode.com/problems/semi-ordered-permutation)

## Description

You are given a **0-indexed** permutation of `n` integers `nums`.

A permutation is called **semi-ordered** if the first number equals `1` and the last number equals `n`. You can perform the below operation as many times as you want until you make `nums` a **semi-ordered** permutation:

- Pick two adjacent elements in `nums`, then swap them.

Return _the minimum number of operations to make_ `nums` _a **semi-ordered permutation**_.

A **permutation** is a sequence of integers from `1` to `n` of length `n` containing each number exactly once.

**Example 1:**

**Input:** nums = \[2,1,4,3\]  
**Output:** 2  
**Explanation:** We can make the permutation semi-ordered using these sequence of operations:  
1 - swap i = 0 and j = 1. The permutation becomes \[1,2,4,3\].  
2 - swap i = 2 and j = 3. The permutation becomes \[1,2,3,4\].  
It can be proved that there is no sequence of less than two operations that make nums a semi-ordered permutation.

**Example 2:**

**Input:** nums = \[2,4,1,3\]  
**Output:** 3  
**Explanation:** We can make the permutation semi-ordered using these sequence of operations:  
1 - swap i = 1 and j = 2. The permutation becomes \[2,1,4,3\].  
2 - swap i = 0 and j = 1. The permutation becomes \[1,2,4,3\].  
3 - swap i = 2 and j = 3. The permutation becomes \[1,2,3,4\].  
It can be proved that there is no sequence of less than three operations that make nums a semi-ordered permutation.

**Example 3:**

**Input:** nums = \[1,3,4,2,5\]  
**Output:** 0  
**Explanation:** The permutation is already a semi-ordered permutation.

**Constraints:**

- `2 <= nums.length == n <= 50`
- `1 <= nums[i] <= 50`
- `nums is a permutation.`

## My Solution

**JavaScript**

```js
// Approach: Determine how far away 1 is from index 0, and how far n is
// from index n - 1. These represent the number of swaps needed to put each number
// in place. There is a special case, however: If n appears before 1, a (simulated) swap
// that brings 1 to index 0 would move n one index closer to n. Thus we subtract 1 from the result.
const semiOrderedPermutation = (nums) => {
  for (let i = 0; i < nums.length; ++i) {
    if (nums[i] === 1) idxMin = i;
    if (nums[i] === nums.length) idxMax = i;
  }

  const numMovesA = idxMin;
  const numMovesB = nums.length - 1 - idxMax;

  if (idxMax < idxMin) return numMovesA + numMovesB - 1;
  return numMovesA + numMovesB;
};
```

| Time Complexity | Space Complexity |
| --------------- | ---------------- |
| O(N)            | O(1)             |

**Python**

```python
def semiOrderedPermutation(self, nums: List[int]) -> int:
    n = len(nums)
    idx_min = -1
    idx_max = -1

    for i in range(n):
        if nums[i] == 1: idx_min = i
        elif nums[i] == n: idx_max = i

    num_moves_min = idx_min
    num_moves_max = len(nums) - 1 - idx_max

    if idx_max < idx_min: return num_moves_min + num_moves_max - 1
    return num_moves_min + num_moves_max
```
